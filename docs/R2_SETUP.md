# Enable R2 image storage (one-time)

Cover images are generated by the art-director agent and, once R2 is configured,
uploaded to object storage so the **URL** (not megabytes of base64) is stored in
the database. Until you complete these steps, images keep working as data URLs —
nothing breaks, the DB just carries more weight.

## Steps (Cloudflare dashboard)
1. **Enable R2**: dash.cloudflare.com → R2 → "Enable R2" (accept terms; free tier covers this scale).
2. **Create a bucket** named `elite-writer-media`.
3. **Make it public** (R2 → the bucket → Settings → Public access → allow), or attach a
   custom domain like `media.insightprofit.live`. Copy the public base URL.
4. **Create an API token**: R2 → "Manage API Tokens" → Create (Object Read & Write).
   Copy the **Access Key ID** and **Secret Access Key**.
5. Find your **Account ID** (R2 overview page, right sidebar).

## Add to the server `.env` (/opt/elite-writer-v5/.env), then `docker compose restart`:
```
R2_ACCOUNT_ID=<account id>
R2_ACCESS_KEY_ID=<access key id>
R2_SECRET_ACCESS_KEY=<secret access key>
R2_BUCKET=elite-writer-media
R2_PUBLIC_URL=https://media.insightprofit.live   # or the bucket's r2.dev URL
```
That's it — the next cover image generated will return an R2 URL (response `storage: "r2"`).
